JavaML: A Markup Language for Java Source Code

Greg J. Badros
University of Washington

Abstract

The typical plain-text representation of source code is convenient for programmers but requires parsing to uncover the deep structure of the program. While sophisticated software tools parse source code to gain access to the program's structure, many lightweight programming aids such as grep rely instead on only the lexical structure of source code. I describe a new XML application that provides an isomorphic alternative representation of Java source code. This XML-based representation, called JavaML, is more natural for tools and permits easy specification of numerous software-engineering analysis by leveraging the plethora of XML tools and techniques. A robust converter built inside the Jikes Java compiler framework translates from plain-text Java source code to JavaML. I argue that the plain-text representation should be viewed as only artifact of the interactive program-editing process, and that JavaML is a better primary representation format for Java programs.
